## Defects surfaced + fixed
D22 — **HIGH**  
Symptom: EP15 had credential helper APIs but no safeStorage-compatible backend implementation, so production host code had no concrete way to satisfy the "secure storage, never plaintext on disk" path.  
Root cause: `packages/core/src/settings/secret-storage.ts` defined the `SecretStorage` contract, and `library-credentials.ts` consumed it, but no Electron-safeStorage adapter existed in the workspace.  
Latency: Introduced with the first EP15 implementation. Existing tests used an in-memory fake and therefore validated helper behavior but not the production encrypted-handle boundary.  
Evidence: `rg "safeStorage|ElectronSafeStorage"` previously found only comments and tests; no implementation.  
Fix PR(s): Fixed directly in this pass by adding `packages/core/src/settings/electron-safe-storage.ts:10` and exporting it from `packages/core/src/settings/index.ts:9`.  
Verification: `packages/core/src/settings/electron-safe-storage.test.ts:17` asserts stored handles do not contain the raw credential and reveal correctly; unavailable safeStorage rejects store/reveal.
